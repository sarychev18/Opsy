**Celery** — Celery — это **фреймворк распределённой очереди задач**. Он не работает в основном потоке, а запускает отдельные `worker`-процессы, которые берут задачи из брокера

## Основные возможности Celery

- **Асинхронное выполнение задач.** Задачи могут выполняться в фоновом режиме, что повышает отзывчивость приложения и позволяет распределять нагрузку между несколькими процессами или машинами. 
- **Распределение нагрузки.** Система может состоять из нескольких воркеров (рабочих процессов), которые извлекают задачи из очереди и выполняют их. 
- **Планирование периодических задач.** Celery поддерживает планирование задач по расписанию (cron-подобные задачи) с помощью модуля 
    celery.beat
- **Построение конвейеров задач.** Можно создавать цепочки, группы и параллельное выполнение задач.
- **Обработка ошибок.** Встроенные механизмы включают автоматические повторные попытки выполнения задачи при сбоях, логирование и уведомления.
- **Гибкая настройка.** Многие компоненты Celery можно расширять или использовать отдельно: пользовательские пулы, сериализаторы, схемы сжатия, планировщики и др.. 

## Ключевые компоненты архитектуры Celery

- **Задача (task)** — обычная Python-функция, «обёрнутая» Celery. Например, 
    send_email  ,    generate_report
- **Брокер сообщений (message broker)** — промежуточное хранилище для задач, которое действует как буфер между продюсером (частью приложения, создающей задачи) и воркером. Поддерживает различные варианты, например Redis, RabbitMQ, Amazon SQS. 
- **Воркер (worker)** — процесс, который слушает брокер, получает задачи из очереди и выполняет их.
- **Бэкенд результатов (result backend)** — хранилище для результатов выполнения задач (успех/ошибка, возвращаемое значение). Может быть Redis, база данных и др..
- **Beat** — опциональный планировщик периодических задач. 

## Примеры задач, которые можно решать с помощью Celery:

- отправка email/SMS/уведомлений;
- генерация отчётов;
- обработка изображений/видео;
- интеграция с внешними API, которые работают медленно;
- переработка больших объёмов данных.

## Установка и использование

Celery можно установить через pip. Для работы требуется указать брокер сообщений и, опционально, бэкенд результатов.

**Пример создания задачи:**

`from celery import Celery  app = Celery('tasks', broker='pyamqp://guest@localhost//')  @app.task def add(x, y):     return x + y`

**Для запуска воркера** используется команда, например: 

celery -A tasks worker --loglevel=info
**Чтобы отправить задачу на выполнение** из основного приложения, можно использовать методы 

task.delay()

 или 

task.apply_async()

```python
from celery import Celery

app = Celery('tasks', broker='pyamqp://guest@localhost//')

@app.task
def add(x, y):
    return x + y
```

Celery используется в производственных системах, например в Instagram, для обработки миллионов задач ежедневно.

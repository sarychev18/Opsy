### 3. Пошаговый план изучения (Roadmap на 6-8 недель)

#### Этап 1: Фундамент и Быстрые победы (Недели 1-2)
*   **Теория:** Архитектура Transformer (статья "Attention is All You Need" — прочитать, понять концепцию, код писать не надо). Tokenization, Context Window, Inference parameters (Temperature, Top-K, Top-P).
*   **Практика:**
    *   Продвинутый Prompt Engineering (Zero-shot, Few-shot, Chain-of-Thought, ReAct).
    *   Работа с API: OpenAI, Anthropic, Groq (для скорости). Напиши скрипты на Python с асинхронными вызовами (твой опыт тут поможет).
    *   Инструменты: Изучи **LangChain** или **LlamaIndex**. Сделай простой чат с документом (PDF), используя их готовые обертки.
*   **Результат:** CLI-приложение или простой FastAPI эндпоинт, который отвечает на вопросы по загруженному тексту.

#### Этап 2: Ядро современной AI-разработки — RAG (Недели 3-4)
*   **Теория:** Как работают эмбеддинги (косинусная близость), проблемы RAG (потеря контекста, "needle in a haystack"), стратегии чанкинга (chunking).
*   **Практика:**
    *   **Векторные БД:** Выбери одну и закопайся в нее. Рекомендую **Qdrant** (написан на Rust, отличный Python-клиент, легко деплоится в Docker) или **Chroma** / **Milvus**.
    *   **Продвинутый RAG:** Реализуй не просто naive RAG, а:
        1.  Разные стратегии чанкинга (Semantic chunking, RecursiveCharacterTextSplitter).
        2.  **Hybrid Search** (Vector + BM25/Full-text search через SQL или встроенные средства БД).
        3.  **Reranking** (использование моделей типа Cohere Rerank или BGE-Reranker для сортировки топ-N результатов).
        4.  Query Transformation (HyDE, Multi-query).
*   **Результат:** Production-ready микросервис на FastAPI + Docker, который делает умный поиск по базе знаний с reranking'ом.

#### Этап 3: Продакшен, Оценка и Безопасность (Недели 5-6)
*   **Теория:** Метрики оценки RAG (Faithfulness, Context Relevance, Answer Relevance). Виды атак на LLM (Prompt Injection, Data Poisoning).
*   **Практика:**
    *   **Evaluation:** Подключи фреймворки для оценки: **Ragas** или **DeepEval**. Напиши скрипт, который оценивает качество ответов твоего RAG на тестовом датасете.
    *   **Observability:** Подключи **LangSmith** или **Arize Phoenix** для трейсинга вызовов и отладки пайплайнов.
    *   **Безопасность:** Попробуй **NeMo Guardrails** или **LLM Guard** для фильтрации входящих/исходящих промптов.
*   **Результат:** Твой RAG-сервис теперь имеет дашборд с метриками качества, логируется и защищен от базовых инъекций.

#### Этап 4: Агенты и Локальные модели (Недели 7-8)
*   **Теория:** Паттерны агентских систем (ReAct, Plan-and-Solve, Multi-Agent). Основы квантования (GGUF, AWQ) и эффективного дообучения (LoRA).
*   **Практика:**
    *   **Агенты:** Изучи **LangGraph** (сейчас это стандарт де-факто для агентов, гораздо лучше старого LangChain Agent). Создай агента, который умеет пользоваться инструментами (Tools): искать в интернете, делать запрос к твоему SQL (Text-to-SQL), отправлять сообщение в Kafka/RabbitMQ.
    *   **Local LLM:** Запусти через **Ollama** или **vLLM** локальную модель (например, Llama-3-8B или Mistral). Подключи свой FastAPI к ней вместо OpenAI.
*   **Результат:** Многоагентная система или продвинутый агент с памятью и инструментами, работающий на локальной/дешевой модели.

### 4. Как использовать твой текущий опыт (Твое конкурентное преимущество)

На собеседованиях и в резюме делай упор на это:
1.  **FastAPI + Docker:** Ты умеешь делать асинхронные, типизированные (Pydantic) и контейнеризированные AI-сервисы. Джуны-дата-саентисты часто пишут скрипты в Jupyter, которые невозможно задеплоить. Ты это умеешь.
2.  **SQL:** Ты можешь делать крутой **Text-to-SQL** RAG или использовать гибридный поиск, опираясь на реляционные данные.
3.  **Kafka/RabbitMQ:** Ты понимаешь, как встроить асинхронную генерацию (Streaming) или обработку длинных AI-задач в микросервисную архитектуру через очереди. Это огромный плюс для Enterprise.

Не трать время на курсы с сертификатами. Сделай 2 крепких проекта на GitHub с отличным README:

1.  **Проект "Enterprise RAG":**
    *   Стек: FastAPI, LangChain/LlamaIndex, Qdrant, PostgreSQL (для метаданных), Ragas (для eval).

2.  **Проект "AI Агент-помощник":**
    *   Стек: LangGraph, Ollama (локальная модель), инструменты (поиск в DuckDuckGo, чтение из твоей же БД).
    *   Фича: Агент, который может выполнить многошаговую задачу (например, "найди последние новости про AI, сделай выжимку и сохрани ее в базу данных").

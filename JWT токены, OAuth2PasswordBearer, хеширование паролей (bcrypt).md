# JWT токены
### 🔹 Для интервью
> «JWT — это стандарт токенов для stateless-аутентификации (Header.Payload.Signature). В Payload кладу только `user_id` и права, **никогда не пароль**. Access Token короткоживущий (15 мин), Refresh Token долгоживущий и хранится в БД для возможности отзыва. Для экстренного бана использую blacklist в Redis.»🔹 Что такое JWT?
**JWT (JSON Web Token)** — открытый стандарт (RFC 7519) для безопасной передачи данных между сторонами. Используется для **stateless-аутентификации**.

### 🔹 Структура токена
```
xxxxx.yyyyy.zzzzz
  │       │       │
Header  Payload  Signature
```
| Часть | Описание | Что внутри |
|-------|----------|------------|
| **Header** | Алгоритм и тип | `{"alg": "HS256", "typ": "JWT"}` |
| **Payload** | Данные (Claims) | `user_id`, `roles`, `exp` (срок). **❌ НЕТ пароля!** |
| **Signature** | Подпись | Проверяет, что токен не подделан (Header + Payload + Secret) |

### 🔹 Жизненный цикл и безопасность

| Компонент | Срок жизни | Где хранить | Можно отозвать? |
|-----------|------------|-------------|-----------------|
| **Access Token** | 5-30 мин | Memory / HttpOnly Cookie | ❌ Нет (stateless), только ждать expiry |
| **Refresh Token** | 7-30 дней | БД / Redis (сервер) | ✅ Да (удалить из БД) |
| **Blacklist** | До expiry токена | Redis (сервер) | ✅ Да (добавить `jti` токена) |

### 🔹 Почему нельзя класть пароль в JWT?
1.  **Payload кодируется, а не шифруется:** Любой может декодировать часть `yyyyy` (base64) и увидеть данные.
2.  **Утечка = компрометация:** Если токен украдут (XSS), злоумышленник получит пароль навсегда.
3.  **Принцип минимизации:** В токене только ID и права, пароль не нужен для авторизации запросов.

### 🔹 Как отозвать доступ?
Так как JWT stateless, сервер не знает о токене пока не проверит подпись.
1.  **Short-lived Access:** Ждать 15 минут пока истечёт.
2.  **Refresh Token Rotation:** При каждом обновлении выдавать новый Refresh, старый удалять.
3.  **Blacklist (Redis):** При logout добавлять `jti` (ID токена) в blacklist до момента `exp`.

### 📘 Мини-термины
| Термин | Определение |
|--------|-------------|
| **Claims** | Утверждения внутри Payload (например, `sub`, `exp`, `iat`). |
| **Stateless** | Сервер не хранит сессию, состояние закодировано в токене. |
| **Signature** | Криптографическая подпись, гарантирующая целостность токена. |
| **jti** | JWT ID — уникальный идентификатор токена для blacklist. |
| **Base64Url** | Способ кодирования частей токена (не шифрование!). |
| **Bearer** | Тип авторизации в заголовке (`Authorization: Bearer <token>`). |

# OAuth2PasswordBearer
### 🔹 Для интервью
> «`OAuth2PasswordBearer` — схема безопасности в FastAPI для реализации OAuth2 Password Flow. Она настраивает Swagger UI для ввода токена и автоматичес
> 
> ки извлекает JWT из заголовка `Authorization: Bearer`. Использую в связке с `Depends` для защиты эндпоинтов и проверки прав пользователя.»
### 🔹 Что такое OAuth2PasswordBearer?
**OAuth2PasswordBearer** — это класс безопасности в **FastAPI** (`fastapi.security`), который реализует схему **OAuth2 Password Flow**. Он сообщает Swagger UI, как клиент должен отправлять токен, и автоматически извлекает его из заголовка запроса.

### 🔹 Как это работает? (Пошагово)
1.  **Настройка:** Создаёшь объект схемы, указывая URL для получения токена (`tokenUrl="token"`).
2.  **Login:** Клиент отправляет `username` и `password` на эндпоинт токена.
3.  **Token:** Сервер возвращает JWT (Access Token).
4.  **Access:** Клиент отправляет JWT в заголовке `Authorization: Bearer <token>`.
5.  **Dependency:** FastAPI автоматически извлекает токен через `Depends(oauth2_scheme)` и передаёт в функцию проверки.

### 🔹 Пример кода
```python
from fastapi import FastAPI, Depends, HTTPException
from fastapi.security import OAuth2PasswordBearer
from jwt import decode, PyJWTError

app = FastAPI()

# ✅ 1. Объявление схемы (для Swagger и извлечения)
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

# ✅ 2. Функция проверки токена (твоя логика)
async def get_current_user(token: str = Depends(oauth2_scheme)):
    try:
        # ✅ Декодирование и проверка подписи (HS256)
        payload = decode(token, SECRET_KEY, algorithms=["HS256"])
        user_id: str = payload.get("sub")
        if user_id is None:
            raise HTTPException(status_code=401, detail="Invalid token")
        return user_id
    except PyJWTError:
        raise HTTPException(status_code=401, detail="Invalid token")

# ✅ 3. Защищённый эндпоинт
@app.get("/users/me")
async def read_users(current_user: str = Depends(get_current_user)):
    return {"user": current_user, "status": "active"}
```

### 🔹 Безопасность и Best Practices
| Аспект | Рекомендация |
|--------|-------------|
| **HTTPS** | Обязательно (токены не должны идти в открытом виде). |
| **Password Hashing** | Пароли в БД должны быть захешированы (bcrypt). |
| **Token Expiry** | Access токены должны жить недолго (15-30 мин). |
| **Scope** | Можно использовать области доступа (scopes) для разделения прав. |
| **Swagger** | Кнопка "Authorize" в docs автоматически использует эту схему. |

### 📘 Мини-термины
| Термин                   | Определение                                                          |
| ------------------------ | -------------------------------------------------------------------- |
| **OAuth2**               | Стандарт авторизации (делегирование доступа).                        |
| **Password Flow**        | Поток, где клиент отправляет логин/пароль напрямую серверу.          |
| **Bearer Token**         | Тип токена, дающий доступ любому обладателю (передается в header).   |
| **Authorization Header** | Заголовок HTTP: `Authorization: Bearer <token>`.                     |
| **Dependency Injection** | Механизм FastAPI (`Depends`) для внедрения логики проверки.          |
| **Swagger UI**           | Интерфейс документации, где кнопка "Authorize" использует эту схему. |
| **Payload**              | Данные внутри JWT (user_id, exp), кодируются в Base64.               |
| **HS256**                | Алгоритм подписи JWT (HMAC with SHA-256).                            |

# хеширование паролей (bcrypt)
### 🔹 Для интервью
> «Для паролей использую **bcrypt** (через `passlib`), так как он медленный и включает соль, что защищает от brute-force и rainbow tables. **Никогда не храню пароли в JWT** — там только ID. Подпись JWT делается через **HMAC/RS256**, а не bcrypt. Cost factor настраиваю так, чтобы хеширование занимало ~200-300мс.»
### 🔹 Что такое bcrypt?
**Bcrypt** — адаптивная функция хеширования паролей. Главное отличие от SHA/MD5 — она **медленная** и **включает соль** по умолчанию.

### 🔹 Зачем нужна соль (Salt)?
**Salt** — случайная строка, добавляемая к паролю **до** хеширования.
1.  **Уникальность:** Даже одинаковые пароли дают разные хэши (у каждого юзера своя соль).
2.  **Защита от Rainbow Tables:** Злоумышленник не может использовать заранее подготовленные таблицы хэшей, так как соль неизвестна.
3.  **Хранение:** Соль хранится **в открытом виде** вместе с хэшем (например, `$2b$12$<salt><hash>`).

### 🔹 bcrypt vs JWT (Важно!)
| Компонент | Алгоритм | Зачем |
|-----------|----------|-------|
| **Пароль в БД** | **bcrypt** | Медленное хеширование для защиты от перебора |
| **Подпись JWT** | **HMAC (HS256)** или **RSA (RS256)** | Быстрая проверка целостности токена |
| **Пароль в JWT** | ❌ **Никогда** | В токене только `user_id` и права |

### 🔹 Пример кода (FastAPI + Passlib)
```python
from passlib.context import CryptContext

# ✅ Настройка контекста (bcrypt по умолчанию)
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# ✅ Хеширование (соль генерируется автоматически)
hashed_password = pwd_context.hash("plain_password")

# ✅ Проверка (соль берется из хэша)
is_valid = pwd_context.verify("plain_password", hashed_password)
```

### 📘 Мини-термины
| Термин | Определение |
|--------|-------------|
| **One-way Function** | Функция, которую легко вычислить, но трудно обратить (хэш → пароль нельзя). |
| **Salt** | Случайные данные, уникальные для каждого хэша, защищают от радужных таблиц. |
| **Rainbow Table** | Таблица предустановленных хэшей для быстрого подбора паролей. |
| **Cost Factor** | Параметр bcrypt (например, 12), определяющий сложность вычисления (2^12 итераций). |
| **Brute-force** | Метод полного перебора всех возможных вариантов пароля. |
| **Passlib** | Популярная библиотека Python для работы с хэшами (обертка над bcrypt). |

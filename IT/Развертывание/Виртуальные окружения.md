## Что такое виртуальное окружение

_**Виртуальное окружение**_ - это изолированная среда Python, которая позволяет устанавливать пакеты отдельно от системной установки Python.

### Зачем нужны виртуальные окружения

**Проблемы без виртуальных окружений:**

- Конфликты версий пакетов между проектами
- Невозможность иметь разные версии одного пакета
- Зависимость от системных пакетов
- Сложность воспроизведения окружения

Изоляция проекта от системных пакетов. Возможность иметь разные версии пакетов в разных проектах. Легкое воспроизведение окружения. Управление зависимостями через requirements.txt.

Виртуальные окружения используются для изоляции проектов, управления зависимостями, обеспечения одинакового окружения для всех разработчиков.

## Создание виртуального окружения

### Создание через venv

_**venv**_ - это встроенный модуль Python для создания виртуальных окружений.

BASH

Копировать

```bash
# Создание виртуального окружения
python -m venv venv

# Структура после создания:
# venv/
#   ├── bin/          # Исполняемые файлы (Linux/macOS)
#   ├── lib/          # Установленные пакеты
#   └── pyvenv.cfg    # Конфигурация окружения
```

Создается папка `venv` с изолированным окружением Python. Все пакеты будут устанавливаться в `venv/lib/`. Окружение изолировано от системной установки Python. Используется для создания изолированного окружения для проекта.

---

## Активация виртуального окружения

### Активация на Linux/macOS

BASH

Копировать

```bash
# Активация виртуального окружения
source venv/bin/activate

# После активации в начале строки появится (venv)
(venv) user@computer:~/SFMShop$
```

Команда `source venv/bin/activate` активирует окружение. В начале строки появляется `(venv)`, показывая активное окружение. Все команды Python и pip теперь используют это окружение. Используется для активации виртуального окружения для работы с проектом.

---

### Активация на Windows

BASH

Копировать

```bash
# Активация виртуального окружения
venv\Scripts\activate

# После активации в начале строки появится (venv)
(venv) C:\Users\user\SFMShop>
```

На Windows используется `venv\Scripts\activate`. Остальное аналогично Linux/macOS.

Используется для активация виртуального окружения на windows.

---

## Деактивация виртуального окружения

BASH

Копировать

```bash
# Деактивация виртуального окружения
deactivate

# После деактивации (venv) исчезнет
user@computer:~/SFMShop$
```

Команда `deactivate` выходит из виртуального окружения. Система возвращается к использованию системного Python.

Используется для выход из виртуального окружения.

## Управление зависимостями

### requirements.txt

_**requirements.txt**_ - это файл, который содержит список всех зависимостей проекта с их версиями.

### Создание requirements.txt

BASH

Копировать

```bash
# После установки всех пакетов в виртуальном окружении
pip freeze > requirements.txt
```

`pip freeze` выводит список всех установленных пакетов с версиями. `> requirements.txt` сохраняет список в файл. Файл содержит все зависимости проекта.

Используется для сохранения списка зависимостей проекта.

---

### Пример requirements.txt

TXT

Копировать

```txt
fastapi==0.104.1
uvicorn==0.24.0
pytest==7.4.3
pytest-cov==4.1.0
psycopg2-binary==2.9.9
pika==1.3.2
redis==5.0.1
pymongo==4.6.0
sqlalchemy==2.0.23
alembic==1.12.1
```

Каждая строка содержит имя пакета и версию. Версии зафиксированы для воспроизводимости. Другой разработчик может установить те же версии.

Используется для управления зависимостями проекта.

---

### Установка зависимостей из requirements.txt

BASH

Копировать

```bash
# Установка всех зависимостей из requirements.txt
pip install -r requirements.txt
```

`pip install -r requirements.txt` устанавливает все пакеты из файла. Устанавливаются именно те версии, которые указаны в файле. Окружение воспроизводится точно так же.

Используется для воспроизведение окружения проекта на другой машине.

---

## Обновление requirements.txt

### Добавление нового пакета

BASH

Копировать

```bash
# 1. Активировать виртуальное окружение
source venv/bin/activate

# 2. Установить новый пакет
pip install requests

# 3. Обновить requirements.txt
pip freeze > requirements.txt
```

Новый пакет устанавливается в виртуальное окружение. `pip freeze` обновляет список всех пакетов. requirements.txt обновляется с новым пакетом.

Используется для добавление новых зависимостей в проект.

## Изоляция проектов

### Разные проекты - разные окружения

BASH

Копировать

```bash
# Проект 1
cd ~/project1
python -m venv venv
source venv/bin/activate
pip install django==4.2

# Проект 2
cd ~/project2
python -m venv venv
source venv/bin/activate
pip install django==3.2
```

Каждый проект имеет свое виртуальное окружение. В разных проектах могут быть разные версии одного пакета. Проекты изолированы друг от друга.

Используется для работа с несколькими проектами с разными зависимостями.

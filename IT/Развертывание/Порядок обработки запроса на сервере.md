#flashcards/IT/Развертывание
Каков порядок обработки запроса на сервере (куда он попадает сначала)?  :: Запрос идёт: **Load Balancer** (распределение, SSL) → **Nginx** (reverse proxy, статика, кэш) → **Application** (Middleware, Auth, Routing, Logic) → **Database** (query, transaction). <br>Load Balancer обеспечивает масштабируемость и отказоустойчивость. Nginx разгружает приложение, отдавая статику и управляя SSL. Middleware обрабатывает запрос до попадания в бизнес-логику (логи, CORS, аутентификация)

### 🔹 Что такое Load Balancer?
**Load Balancer** — устройство или сервис, который распределяет входящие сетевые запросы между несколькими серверами (бэкендами).

| Задача | Как решает |
|--------|-----------|
| **Масштабирование** | Распределяет нагрузку на N серверов (horizontal scaling) |
| **Отказоустойчивость** | Если сервер упал — трафик идёт на другие (health check) |
| **SSL Termination** | Расшифровка HTTPS на балансировщике (разгрузка приложения) |
| **Geo-routing** | Направление пользователя к ближайшему дата-центру |
| **Canary Deploy** | Направление части трафика на новую версию приложения |

### 🔹 Что такое Nginx?
**Nginx** — веб-сервер и reverse proxy, который стоит перед приложением.

| Функция | Описание |
|---------|----------|
| **Reverse Proxy** | Принимает запросы и перенаправляет на приложение (Gunicorn/Uvicorn) |
| **Static Files** | Отдаёт CSS, JS, images напрямую (быстрее, чем через Python) |
| **SSL/HTTPS** | Управление сертификатами (Let's Encrypt) |
| **Rate Limiting** | Ограничение запросов в секунду (защита от DDoS) |
| **Compression** | Gzip/Brotli сжатие ответов |
| **Caching** | Кэширование ответов приложения |

### 📘 Мини-термины
| Термин                 | Определение                                                             |
| ---------------------- | ----------------------------------------------------------------------- |
| **Load Balancer**      | Распределяет трафик между серверами для надёжности и масштаба.          |
| **Reverse Proxy**      | Сервер, который принимает запросы и перенаправляет их приложению.       |
| **Middleware**         | Промежуточный слой в приложении (логи, auth, CORS) до попадания в view. |
| **SSL Termination**    | Расшифровка HTTPS-трафика на балансировщике/Nginx.                      |
| **Health Check**       | Периодическая проверка доступности серверов (ping/endpoint).            |
| **Horizontal Scaling** | Добавление новых серверов для обработки нагрузки.                       |
| **Static Files**       | Статика (CSS, JS, images), которую отдаёт Nginx напрямую.               |
| **WAF**                | Web Application Firewall — защита от атак (SQLi, XSS).                  |
| **CDN**                | Content Delivery Network — кэширование контента ближе к пользователю.   |
| **Connection Pool**    | Пул соединений с БД для повторного использования.                       |
| **Rate Limiting**      | Ограничение количества запросов от одного клиента.                      |
| **Canary Deploy**      | Постепенный rollout новой версии на часть трафика.                      |

### 🔹 Пример конфигурации Nginx
```nginx
# ✅ /etc/nginx/nginx.conf
server {
    listen 80;
    server_name example.com;
    
    # ✅ Статика
    location /static/ {
        alias /app/static/;
    }
    
    # ✅ Проксирование на приложение
    location / {
        proxy_pass http://127.0.0.1:8000;  # Uvicorn/Gunicorn
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

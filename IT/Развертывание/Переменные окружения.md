## Что такое переменные окружения

_**Переменные окружения**_ - это переменные, которые доступны процессу из операционной системы.

### Зачем нужны переменные окружения

**Проблемы без переменных окружения:**

- Конфигурация захардкожена в коде
- Секреты в коде - небезопасно
- Невозможно настроить для разных окружений
- Сложность изменения конфигурации

Безопасное хранение секретов. Настройка для разных окружений. Легкое изменение конфигурации. Разделение кода и конфигурации.

Используется для хранение конфигурации, секретов, настроек для разных окружений.

---

## Использование переменных окружения в Python

### os.getenv()

PYTHON

Копировать

```python
import os

# Получение переменной окружения
database_url = os.getenv("DATABASE_URL")
secret_key = os.getenv("SECRET_KEY")

# Значение по умолчанию
database_url = os.getenv("DATABASE_URL", "postgresql://localhost:5432/sfmshop")
```

`os.getenv()` получает переменную окружения. Второй параметр - значение по умолчанию. Если переменная не установлена, возвращается None или значение по умолчанию.

Используется для получение переменных окружения в python.

---

### os.environ

PYTHON

Копировать

```python
import os

# Получение переменной окружения
database_url = os.environ.get("DATABASE_URL")

# Установка переменной окружения (в коде)
os.environ["DATABASE_URL"] = "postgresql://localhost:5432/sfmshop"
```

`os.environ` - словарь всех переменных окружения. `get()` получает значение с возможностью значения по умолчанию. Можно устанавливать переменные в коде.

Используется для работа с переменными окружения через словарь.

## .env файлы

_**.env файл**_ - это файл с переменными окружения для локальной разработки.

### Создание .env файла

BASH

Копировать

```bash
# .env
DATABASE_URL=postgresql://user:password@localhost:5432/sfmshop
SECRET_KEY=my-secret-key-12345
REDIS_URL=redis://localhost:6379
DEBUG=True
```

Файл `.env` содержит переменные окружения. Каждая строка - переменная в формате KEY=VALUE. Файл не должен попадать в git (добавить в .gitignore).

Используется для хранение переменных окружения для локальной разработки.

---

## Библиотека python-dotenv

_**python-dotenv**_ - это библиотека для загрузки переменных из .env файла.

### Установка python-dotenv

BASH

Копировать

```bash
pip install python-dotenv
```

### Использование python-dotenv

PYTHON

Копировать

```python
from dotenv import load_dotenv
import os

# Загрузка переменных из .env файла
load_dotenv()

# Использование переменных
database_url = os.getenv("DATABASE_URL")
secret_key = os.getenv("SECRET_KEY")
```

`load_dotenv()` загружает переменные из .env файла. После загрузки переменные доступны через `os.getenv()`. Переменные загружаются автоматически при запуске.

Используется для загрузка переменных окружения из.env файла.

## Безопасное хранение секретов

### .env файл для разработки

BASH

Копировать

```bash
# .env (для локальной разработки)
DATABASE_URL=postgresql://user:password@localhost:5432/sfmshop
SECRET_KEY=dev-secret-key
REDIS_URL=redis://localhost:6379
DEBUG=True
```

### .env.example файл

BASH

Копировать

```bash
# .env.example (шаблон для других разработчиков)
DATABASE_URL=postgresql://user:password@localhost:5432/sfmshop
SECRET_KEY=your-secret-key-here
REDIS_URL=redis://localhost:6379
DEBUG=True
```

`.env` содержит реальные значения (не в git). `.env.example` содержит шаблон (в git). Другие разработчики копируют `.env.example` в `.env`.

Используется для безопасное хранение секретов и шаблон для других разработчиков.

## Конфигурация для разных окружений

### Разные .env файлы

BASH

Копировать

```bash
# .env.development
DATABASE_URL=postgresql://localhost:5432/sfmshop_dev
DEBUG=True

# .env.production
DATABASE_URL=postgresql://prod-server:5432/sfmshop
DEBUG=False
```

### Загрузка нужного файла

PYTHON

Копировать

```python
from dotenv import load_dotenv
import os

# Определение окружения
env = os.getenv("ENVIRONMENT", "development")

# Загрузка соответствующего .env файла
load_dotenv(f".env.{env}")

# Использование переменных
database_url = os.getenv("DATABASE_URL")
debug = os.getenv("DEBUG", "False").lower() == "true"
```

Разные .env файлы для разных окружений. Загрузка нужного файла в зависимости от окружения. Конфигурация меняется в зависимости от окружения.

Используется для настройка для разных окружений (dev, staging, production).

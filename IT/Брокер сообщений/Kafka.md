### Consumer groups

#flashcards/брокер_сообщений 
что такое Consumer groups в Kafka? :: Группа потребителей, которые вместе читают сообщения из топика. Каждое сообщение получает только один consumer из группы (для балансировки). Разные группы читают топик независимо.

## Топ-20 вопросов по Apache Kafka для Python Middle

---

### 1. Что такое Apache Kafka? Для чего используется?

**Ответ:** Kafka — распределенная платформа для потоковой обработки событий (event streaming). Хранит потоки данных в топиках, позволяет публиковать, подписываться, обрабатывать и хранить события. Используется для:
- Сбора логов и метрик
- Обмена данными между микросервисами
- Event sourcing
- Stream processing (аналитика в реальном времени)

---

### **2. Чем Kafka отличается от традиционных очередей (RabbitMQ)?**

| Критерий               | Kafka                          | RabbitMQ                 |
| ---------------------- | ------------------------------ | ------------------------ |
| **Модель**             | Pull (consumer тянет)          | Push (брокер отправляет) |
| **Хранение**           | Долгое (7 дней, настраивается) | Удаляет после ack        |
| **Порядок**            | Внутри партиции                | FIFO в очереди           |
| **Производительность** | Миллионы msg/сек               | Тысячи msg/сек           |
| **Consumer groups**    | Да (балансировка)              | Да (work queues)         |

---

### **3. Что такое топик (topic) в Kafka?**

**Ответ:** Топик — это категория/канал для сообщений. Producer пишет в топик, consumer читает из топика. Топик разбивается на партиции (partitions) для параллелизма.

```python
# Создание топика через API
admin.create_topics([
    NewTopic('orders', num_partitions=3, replication_factor=2)
])
```

---

### **4. Что такое партиция (partition) и зачем она нужна?**

**Ответ:** Партиция — это единица параллелизма в Kafka. Сообщения внутри партиции упорядочены (каждое имеет offset). Партиции позволяют:
- Масштабировать (разные партиции на разных брокерах)
- Параллелить обработку (разные consumer'ы читают разные партиции)
- Обеспечивать порядок по ключу

---

### **5. Что такое offset в Kafka?**

**Ответ:** Offset — уникальный номер сообщения внутри партиции. Монотонно возрастает. Consumer запоминает свой offset, чтобы знать, какие сообщения уже прочитаны.

```python
# Каждое сообщение имеет offset
record = consumer.poll()
print(f"Partition: {record.partition}, Offset: {record.offset}, Value: {record.value}")
```

---

### **6. Что такое consumer group?**

**Ответ:** Группа потребителей, которые вместе читают сообщения из топика. Каждая партиция отдается **только одному consumer'у** из группы. Позволяет:
- Балансировать нагрузку
- Масштабировать обработку
- Гарантировать порядок внутри партиции

```python
# Consumer в группе
consumer = KafkaConsumer('orders', group_id='payment-processor', bootstrap_servers='...')
```

---

### **7. Как Kafka гарантирует порядок сообщений?**

**Ответ:** Порядок гарантируется **внутри одной партиции**. Сообщения с одинаковым ключом попадают в одну партицию. Глобальный порядок невозможен без одной партиции (теряем параллелизм).

```python
# Сообщения с ключом user123 всегда в одной партиции → будут по порядку
producer.send('orders', key='user123', value='order1')
producer.send('orders', key='user123', value='order2')
```

---

### **8. Что такое репликация (replication) в Kafka?**

**Ответ:** Копирование партиций на несколько брокеров для отказоустойчивости. У каждой партиции есть лидер (leader) и несколько фолловеров (followers). При падении лидера фолловер становится новым лидером.

```python
# Создание топика с репликацией
NewTopic('critical', num_partitions=3, replication_factor=3)
```

---

### **9. Что такое ISR (In-Sync Replicas)?**

**Ответ:** ISR — список реплик, которые полностью синхронизированы с лидером. Producer может настроить `acks`, чтобы ждать подтверждения от всех ISR. Чем больше ISR, тем надежнее, но медленнее.

```python
# Producer ждет подтверждения от всех ISR
producer = KafkaProducer(acks='all', bootstrap_servers='...')
```

---

### **10. Что такое acks в Kafka?**

**Ответ:** Настройка подтверждения записи от брокера:
- **acks=0** — producer не ждет (быстро, возможна потеря)
- **acks=1** — ждет подтверждения от лидера (надежнее)
- **acks=all** — ждет подтверждения от всех ISR (самое надежное)

---

### **11. Что такое retention policy в Kafka?**

**Ответ:** Правила хранения сообщений. Можно настроить:
- **По времени** (retention.ms) — удалять сообщения старше N дней
- **По размеру** (retention.bytes) — удалять, если топик превысил размер
- **Compact policy** — хранить только последнее значение по ключу

---

### **12. Что такое compaction в Kafka?**

**Ответ:** Лог-компакция — механизм, при котором в топике хранится только последнее сообщение для каждого ключа. Используется для хранения текущего состояния (например, последний баланс пользователя).

```python
# Топик с компакцией
config = {'cleanup.policy': 'compact'}
NewTopic('user-balances', num_partitions=3, config=config)
```

---

### **13. Что такое Kafka Streams?**

**Ответ:** Библиотека для stream processing — обработки потоков данных в реальном времени. Позволяет делать агрегации, джойны, оконные операции прямо в Kafka.

```java
// Пример на Java (для Python используют Faust или kafka-python)
KStream<String, Order> stream = builder.stream("orders");
stream.groupByKey().count().toStream().to("order-counts");
```

---

### **14. Что такое ZooKeeper и зачем он нужен Kafka?**

**Ответ:** ZooKeeper — координационный сервис для управления кластером Kafka (до версии 2.8). Хранит метаданные: кто лидер, какие брокеры живы, конфигурации топиков. С Kafka 2.8+ появился KRaft (без ZooKeeper).

---

### **15. Что такое KRaft в Kafka?**

**Ответ:** Режим работы Kafka без ZooKeeper (начиная с версии 2.8). Управление кластером через внутренний Raft-консенсус. Упрощает архитектуру и улучшает производительность.

---

### **16. Как обеспечить exactly-once семантику в Kafka?**

**Ответ:** Exactly-once требует:
1. Идемпотентный producer (`enable.idempotence=true`)
2. Транзакционный producer (`transactional.id`)
3. Consumer с изоляцией read_committed

```python
producer = KafkaProducer(
    bootstrap_servers='...',
    enable_idempotence=True,
    transactional_id='tx-123'
)
producer.begin_transaction()
# отправка сообщений
producer.commit_transaction()
```

---

### **17. Что такое rebalance в consumer group?**

**Ответ:** Перераспределение партиций между consumer'ами при:
- Добавлении/удалении consumer'а
- Добавлении партиций
- Падении consumer'а

Во время rebalance обработка сообщений приостанавливается.

---

### **18. Как работает механизм pull в Kafka?**

**Ответ:** Kafka использует модель pull — consumer сам запрашивает сообщения, когда готов. Это отличается от push-модели RabbitMQ. Плюсы: consumer не перегружается, может контролировать скорость.

---

### **19. Что такое Kafka Connect?**

**Ответ:** Инструмент для интеграции Kafka с внешними системами (БД, S3, Elasticsearch). Source connectors импортируют данные в Kafka, sink connectors экспортируют из Kafka.

---

### **20. Какие типичные проблемы Kafka и их решения?**

| Проблема                    | Решение                                                 |
| --------------------------- | ------------------------------------------------------- |
| **Сообщения не по порядку** | Использовать один ключ для связанных событий            |
| **Duplicate messages**      | Идемпотентный producer + дедупликация в consumer        |
| **Consumer lag**            | Увеличить партиции, добавить consumer'ов                |
| **Потеря данных**           | `acks=all`, replication.factor=3, min.insync.replicas=2 |
| **Rebalance слишком часто** | Настроить session.timeout.ms, heartbeat.interval.ms     |

---

## **БОНУС: Пример producer/consumer на Python**

```python
from kafka import KafkaProducer, KafkaConsumer
import json

# Producer
producer = KafkaProducer(
    bootstrap_servers=['localhost:9092'],
    value_serializer=lambda v: json.dumps(v).encode('utf-8'),
    acks='all',
    enable_idempotence=True
)
producer.send('orders', key=b'123', value={'order_id': 123, 'amount': 100})
producer.flush()

# Consumer
consumer = KafkaConsumer(
    'orders',
    bootstrap_servers=['localhost:9092'],
    group_id='payment-group',
    value_deserializer=lambda m: json.loads(m.decode('utf-8')),
    auto_offset_reset='earliest'  # с самого начала, если нет offset
)
for msg in consumer:
    print(f"Key: {msg.key}, Value: {msg.value}, Partition: {msg.partition}, Offset: {msg.offset}")
```

### Что такое AMQP?

#flashcards/брокер_сообщений
Что такое AMQP?  :: Advanced Message Queuing Protocol — открытый протокол для обмена сообщениями. Определяет:<br>• **Exchange** — получает сообщения от producer<br>• **Queue** — хранит сообщения<br>• **Binding** — связь exchange с queue<br>• **Virtual Host** — изоляция окружений

в RabbitMQ это самая популярная реализация AMQP.

----------------------------------------------------
### **Exchange в RabbitMQ? Типы exchange.**

#flashcards/брокер_сообщений
Что такое Exchange в RabbitMQ? Типы exchange.  :: Exchange — компонент, который получает сообщения от producer и распределяет их в очереди по правилам (bindings).<br><br>**Типы exchange:**<br>• **Direct** — точное совпадение routing key<br>• **Fanout** — всем подключенным очередям (как broadcast)<br>• **Topic** — по шаблону (wildcards: `*` и `#`)<br>• **Headers** — по заголовкам (не по routing key)

```python
# Direct
channel.queue_bind(exchange='direct_logs', queue='error', routing_key='error')
# Fanout
channel.exchange_declare(exchange='logs', exchange_type='fanout')
```

### **Очередь (Queue) и чем отличается от топика (Topic)?**

#flashcards/брокер_сообщений
что такое очередь (Queue) и чем отличается от топика (Topic)?  ::<br>• **Queue** — хранит сообщения, каждый consumer получает каждое сообщение один раз (work queue). Сообщения удаляются после обработки.<br>• **Topic** (в Kafka/PubSub) — хранит поток сообщений, несколько consumer групп могут читать независимо, каждое сообщение хранится долго.

### **Как обеспечивается надежность (durability) в RabbitMQ?**

#flashcards/брокер_сообщений
Что такое Exchange в RabbitMQ? Типы exchange.  ::<br>**Три уровня:**<br>• **Queue durable** — очередь не исчезает после перезапуска<br>• **Message persistent** — сообщение сохраняется на диск<br>• **Publisher confirms** — producer получает подтверждение, что сообщение сохранено
    

```python

# Durable queue
channel.queue_declare('tasks', durable=True)
# Persistent message
channel.basic_publish(
    exchange='', 
    routing_key='tasks', 
    body='task',
    properties=pika.BasicProperties(delivery_mode=2)  # persistent
)
# Publisher confirm
channel.confirm_delivery()
channel.basic_publish(...)  # Бросает исключение при ошибке
```

### **Что такое Federation и Shovel в RabbitMQ?**

#flashcards/брокер_сообщений
Что такое Exchange в RabbitMQ? Типы exchange.  :: <br>• **Federation** — связь брокеров для обмена сообщениями между кластерами (публикация в одном → получают все)<br>• Shovel** — пересылка сообщений из одной очереди в другую (возможно, между разными брокерами)<br><br>**Нужны для распределенных систем и кросс-региональной репликации.**

    # PRODUCER
    import aio_pika
    import json
    
    async def publish_order(order_data):
        connection = await aio_pika.connect_robust("amqp://guest:guest@localhost/")
        channel = await connection.channel()
        
        exchange = await channel.declare_exchange(
            'orders', aio_pika.ExchangeType.TOPIC, durable=True
        )
        
        message = aio_pika.Message(
            body=json.dumps(order_data).encode(),
            delivery_mode=aio_pika.DeliveryMode.PERSISTENT,
            message_id=str(uuid.uuid4())
        )
        
        await exchange.publish(message, routing_key='order.created')
        await connection.close()

    ───────────────────────────────────────────────────────────────────────────

    # CONSUMER
    import aio_pika
    import json
    
    async def on_message(message: aio_pika.IncomingMessage):
        async with message.process():  # auto ACK при успехе
            data = json.loads(message.body)
            try:
                await process_order(data)  # бизнес-логика
            except Exception:
                await message.nack(requeue=True)  # вернётся в очередь
    
    async def consume():
        connection = await aio_pika.connect_robust("amqp://guest:guest@localhost/")
        channel = await connection.channel()
        await channel.set_qos(prefetch_count=1)
        
        queue = await channel.declare_queue('orders_q', durable=True)
        await queue.consume(on_message)

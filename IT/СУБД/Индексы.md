Расскажите какие виды и типы индексов вы знаете и расскажите про случаи, когда требуется применять каждый из них?  :: Типы индексов PostgreSQL и их применение:<br>**Тип**      | Структура                              | 	Операторы                                      |	Когда использовать<br>**B-tree**   |Сбалансированное дерево   |	=, >, <, BETWEEN, LIKE 'prefix%'	 |Основной тип, сортировка (ORDER BY), диапазоны<br>**Hash**     | Хэш-таблица                         | =	Только точное равенство          | (редко используется)<br>**GiST**      | Дерево для сложных типов | 	@>, <@, &&, ~	                         |Геоданные, полнотекст, диапазоны (range)<br>**GIN**       | Обратный индекс                 |	@>, @?, @@	                                 |Массивы, JSON, полнотекстовый поиск<br>**BRIN**     | Блочный индекс                   |	=, <, >	                                             |Большие таблицы с естественной сортировкой (логи, метрики)<br><br>**Дополнительные возможности:**<br>✅ **Составной индекс:** `CREATE INDEX idx ON t (a, b)` — важен порядок<br>✅ **Частичный индекс:** `CREATE INDEX idx ON t (a) WHERE b > 0` — только подмножество строк<br>✅ **UNIQUE индекс:** автоматически создаётся при `UNIQUE CONSTRAINT`

```SQL
-- B-tree (основной случай)
CREATE INDEX idx_users_created ON users(created_at);

-- GIN (поиск в массиве)
CREATE INDEX idx_products_tags ON products USING GIN (tags);
```

#flashcards/СУБД 
Что такое покрывающий индекс?  :: **Покрывающий индекс** — это индекс, который **включает все столбцы SELECT-запроса**, позволяя выполнять запрос только через индекс (без чтения таблицы), что снижает I/O операции. Создается с помощью `CREATE INDEX ... INCLUDE`.<br><br>**Уточнения:**<br>✅ цель — оптимизация SELECT, экономия I/O
<!--SR:!2026-04-24,17,228-->

```SQL
-- Запрос: SELECT name, email FROM users WHERE age > 25;
CREATE INDEX idx_users_age_covering ON users (age) INCLUDE (name, email);
```

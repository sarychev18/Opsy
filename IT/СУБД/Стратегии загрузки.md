2. **КРАТКИЙ ОТВЕТ (для интервьюера):**
   - «Проблема **N+1** возникает, когда мы делаем 1 запрос для получения списка объектов и ещё N запросов для доступа к их связям внутри цикла.
   - **Решение:** Eager Loading (жадная загрузка).
   - **`joinedload`:** Делает `JOIN` в одном запросе. Хорошо для `One-to-One` или `Many-to-One`.
   - **`selectinload`:** Делает 2 запроса (второй через `IN`). Лучше для `One-to-Many`, чтобы избежать дублирования строк (Cartesian Product) 🚀».

3. **ПОЛНЫЙ ОТВЕТ (для глубокого понимания):**

   **👵 Как для бабушки:**
   Представь, что ты заказываешь еду для группы друзей (список пользователей).
   - **N+1 (Плохо):** Ты спрашиваешь каждого друга отдельно: «Что ты хочешь?» (1 вопрос другу + N вопросов каждому). Долго и глупо.
   - **`joinedload` (Вместе):** Ты сразу приносишь общий стол с едой для всех. Всё в одной куче.
   - **`selectinload` (Отдельно но быстро):** Ты сначала собираешь список друзей, а потом одним вопросом спрашиваешь кухню: «Принесите блюда для Васи, Пети и Оли!».
   - **Зачем выбирать:** Если блюд много у каждого, общий стол (`joinedload`) будет слишком большим и неудобным. Лучше отдельный запрос (`selectinload`) 🍽️.

   **👨‍💻 Для глубокого понимания (Technical Deep Dive):**
   Для Middle разработчика важно видеть SQL-запросы, которые генерирует ORM.

   **1. Проблема N+1 (The Problem):**
   ```python
   # ❌ ПЛОХО: N+1 запросов
   users = session.query(User).all()  # 1 запрос
   for user in users:
       print(user.orders)  # +1 запрос на каждого пользователя (N запросов)
   # Итого: 1 + N запросов. При 100 юзерах = 101 запрос к БД! 📉
   ```

   **2. Решение: `joinedload` (JOIN):**
   ```python
   # ✅ ХОРОШО: 1 запрос через JOIN
   from sqlalchemy.orm import joinedload
   
   users = session.query(User).options(joinedload(User.orders)).all()
   # SQL: SELECT * FROM users LEFT JOIN orders ON users.id = orders.user_id
   ```
   - **Минус:** Если у пользователя 100 заказов, строка пользователя повторится 100 раз в результате (Cartesian Product). Это грузит сеть и память.

   **3. Решение: `selectinload` (IN clause):**
   ```python
   # ✅ ЛУЧШЕ для One-to-Many: 2 запроса
   from sqlalchemy.orm import selectinload
   
   users = session.query(User).options(selectinload(User.orders)).all()
   # Запрос 1: SELECT * FROM users
   # Запрос 2: SELECT * FROM orders WHERE user_id IN (1, 2, 3...)
   ```
   - **Плюс:** Нет дублирования данных пользователей. Часто быстрее на больших данных.

   **4. Когда что использовать (Cheat Sheet):**
   - **`joinedload`:** Связи `Many-to-One` (Заказ → Пользователь) или `One-to-One`.
   - **`selectinload`:** Связи `One-to-Many` (Пользователь → Заказы) или `Many-to-Many`.
   - **`lazy='select'`:** (По умолчанию) — это и есть причина N+1. Избегай в критичных эндпоинтах.

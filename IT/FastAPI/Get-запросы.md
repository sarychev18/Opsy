ПРАВИЛЬНЫЙ ОТВЕТ (кратко для интервьюера):

Как работает GET-запрос:

1. Назначение: идемпотентный, безопасный метод HTTP для получения ресурса с сервера (не изменяет данные)

2. Передача параметров:
   - Path-параметры — идентификатор ресурса в URL (`/users/42`)
   - Query-параметры — фильтрация, пагинация после `?` (`/users?age=25&page=2`)

3. Структура запроса:

   ```
   GET /api/books/1?fields=title,author HTTP/1.1
   Host: example.com
   Accept: application/json
   ```

4. Обработка в FastAPI:

   ```python
   from fastapi import FastAPI, Query, Path
   app = FastAPI()
   
   @app.get("/books/{book_id}")  # Path-параметр
    def get_book(
       book_id: int = Path(..., description="ID книги"),
       fields: str = Query(None, description="Какие поля вернуть") # Query-параметр
   ):

       # Логика получения книги из БД
       book = db.get_book(book_id)
       if not book:
           return JSONResponse(status_code=404, content={"error": "Not found"})
       return {"id": book_id, "fields": fields}
   ```

5. Коды ответа:
   - 200 OK — ресурс найден и возвращён
   - 404 Not Found — ресурс не существует
   - 400 Bad Request — некорректные параметры
   - 500 Internal Server Error — ошибка на стороне сервера

Ключевые особенности:
- ✅ Не имеет тела (body) — все данные в URL или заголовках
- ✅ Идемпотентен — повторные запросы не меняют состояние
- ✅ Кэшируется — может быть закэширован браузером/CDN

Идемпотентный метод HTTP-запроса — это метод, при котором повторные идентичные запросы имеют тот же эффект, что и один запрос. Другими словами, многократное выполнение идемпотентной операции приводит к тому же результату, что и однократное. 

Идемпотентный метод не должен иметь никаких побочных эффектов (side-effects), кроме сбора статистики или подобных операций. 

Виды
В REST идемпотентными считаются следующие HTTP-методы: 

GET — многократное получение ресурса не изменяет состояние сервера.
PUT — многократное обновление ресурса одними и теми же данными приводит к одному и тому же результату.

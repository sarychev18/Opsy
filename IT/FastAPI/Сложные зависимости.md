2. **КРАТКИЙ ОТВЕТ (для интервьюера):**
   - «В FastAPI «сложные зависимости» — это не просто функции, а **классы с `__call__`**, **вложенные графы зависимостей** (`Depends` внутри `Depends`) и **генераторы** (`yield`) для управления жизненным циклом.
   - Они позволяют инкапсулировать логику (например, авторизацию + получение юзера из БД) в переиспользуемые компоненты.
   - FastAPI автоматически разрешает граф зависимостей и кэширует результат в рамках одного запроса 🚀».

   **1. Классы как зависимости (Stateful Dependencies):**
   Если зависимость должна хранить состояние (например, контекст пользователя), используют класс.
   ```python
   from fastapi import Depends, HTTPException
   
   class CurrentUser:
       def __init__(self, token: str = Depends(oauth2_scheme)):
           self.user = get_user_from_token(token)
           if not self.user:
               raise HTTPException(401)
       
       def __call__(self):
           return self.user
   
   @app.get("/me")
   def read_me(current: CurrentUser = Depends()):
       return current  # Вызывается __call__
   ```

   **2. Вложенные зависимости (Dependency Graph):**
   Зависимость может сама иметь зависимости. FastAPI строит граф и решает порядок выполнения.
   ```python
   def get_db():  # Уровень 1
       db = SessionLocal()
       yield db
       db.close()
   
   def get_repo(db: Session = Depends(get_db)):  # Уровень 2
       return UserRepository(db)
   
   @app.get("/users")
   def get_users(repo: UserRepository = Depends(get_repo)):  # Уровень 3
       return repo.find_all()
   ```
   - **Порядок:** `get_db` → `get_repo` → `get_users`.
   - **Очистка:** В обратном порядке (сначала `get_users`, потом `db.close()`).

   **3. Кэширование зависимостей:**
   Если одна зависимость (`get_db`) используется в нескольких местах одного запроса, она выполнится **один раз**.
   ```python
   # Если оба эндпоинта вызываются в рамках одного запроса (редко, но бывает в подзапросах)
   # FastAPI переиспользует объект DB, а не создает новый.
   ```

   **4. Зачем это нужно (Middle Level):**
   - **DRY:** Логика авторизации пишется один раз в классе `CurrentUser`.
   - **Testing:** Легко подменить весь класс через `dependency_overrides`.
   - **Security:** Защита встроена в сигнатуру функции, нельзя забыть проверить права.

   **🎯 Совет для Middle:**
   На собеседовании скажи: «Я использую классы-зависимости для группировки логики авторизации и доступа к данным. Это позволяет мне держать эндпоинты чистыми и тестировать бизнес-логику отдельно от транспорта».

   Ответ: В-Б-А, yield в В

Декораторы это функции высшего порядка, они используются для добавления функциональности функциям, не изменяя их код. 

Допустим есть такая функция.
```python
def lubaya_funkcia():
    print('My name is Evgenii')

lubaya_funkcia()
```

Она печатает такой текст
```html
'My name is Evgenii'
```
Чтобы добавить функционал к этой функции можно написать декоратор
```python
def my_decorator(func):
    def inner():
        print('дополнил функцию этим сообщением')
        func()
        print('дополнил функцию еще и этим сообщением')
    return inner

@my_decorator
def lubaya_funkcia():
    print('My name is Evgenii')

lubaya_funkcia()
```
и тогда результатом вызова функции lubaya_funkcia() будет:
```HTML
дополнил функцию этим сообщением
My name is Evgenii
дополнил функцию еще и этим сообщением
```
Вот подробное объяснение как написать декоратор
```python
def my_decorator(func): #Создаём функцию (она будет принимать одну, а возвращать другую функцию)
    def inner(): #Создаём функцию (где будет выполняться обычная функция с дополнением)
        print('дополнил функцию этим сообщением') #дополняется
        func() #исполняется обычная функция
        print('дополнил функцию еще и этим сообщением') #дополняется
    return inner #возвращается другая функция

@my_decorator #@ - используется для вызова декоратора
def lubaya_funkcia():
    print('My name is Evgenii')

lubaya_funkcia() 
```
Это был декоратор без аргументов
# Декораторы с параметрами
для декораторов с переменными используется [[Args_and_Kwargs]] 
Общая структура выглядит так:
```python
def decorator_with_params(param):
    def actual_decorator(func):
        def wrapper(*args, **kwargs):
            # Логика декоратора
            return func(*args, **kwargs)
        return wrapper
    return actual_decorator
```

- **param** — это параметр, который передаётся в декоратор.
- **actual_decorator** — настоящая функция-декоратор, которая принимает оригинальную функцию (`func`).
- **wrapper** — функция-обёртка, которая добавляет новое поведение.
Принцип работы декоратора с параметрами:
1. Сначала создаем функцию верхнего уровня(сообщение или изменяемый параметр)
2. Создаём функцию-декоратор(принимает основную функцию)
3. Создаём функцию - обертку(принимает \*args, \*\*kwargs)

Пример
```python
#мы хотим измерять время выполнения функции и позволить пользователю выбирать единицы измерения времени (секунды, миллисекунды или микросекунды).
import time

def timer(unit='ms'): #функция верхнего уровня(сообщение или изменяемый параметр)
    def decorator(func): #функция-декоратор(принимает основную функцию)
        def wrapper(*args, **kwargs): #функция - обертка(принимает *args, **kwargs)
            start_time = time.perf_counter() #время до начала программы(мс)
            result = func(*args, **kwargs) #запуск программы
            end_time = time.perf_counter() #время завершения программы(мс)
            elapsed_time = (end_time - start_time) * 1000  # миллисекунды по умолчанию
            
            if unit == 'sec': 
                elapsed_time /= 1000
            elif unit == 'us':
                elapsed_time *= 1000
            
            print(f"Функция {func.__name__} выполнена за {elapsed_time:.2f} {unit}")
            return result #возращаем результат основной программы
        return wrapper #возращаем результат функции обертки
    return decorator #возращаем результат функции декоратора - функции обертки

@timer(unit='sec')
def some_heavy_calculation():
    total = 0
    for i in range(1000000):
        total += i
    return total

some_heavy_calculation()
```

## Декораторы в ООП
В ООП есть декораторы, которые меняют методы или атрибуты классов, не меняя их исходный код

@classmethod - работает с атрибутами класса (ссылается на класс )
 
```python
class Person:
	count = 0

	def __init__(self,name):
		self.name = name
		Person.count += 1
		
	@classmethod #декоратор ссылается на атрибут класа через cls
	def get_count(cls):
		return cls.count


# Использование
print(Person.get_count())  # Вывод: 0
p1 = Person("Alice")
p2 = Person("Bob")
print(Person.get_count())  # Вывод: 2
```

@staticmethod - не связан ни с классом, ни с экземпляром. Он не принимает ни `self`, ни `cls` в качестве первого аргумента.
 
```python
class Math:
	@staticmethod
	def add(a,b):
		return a + b

# Использование
print(Math.add(2, 3))  # Вывод: 5
```
@staticmethod нужен для то чтобы делать независимые функции внутри класса, улучшить читаемость кода, использовать их в качестве вспомогательных функций

@property -  позволяет обращаться к методу класса как к атрибуту
 
```python
class Circle:
    def __init__(self, radius):
        self._radius = radius  #_radius инкапсуляция для защиты и сохранения значения

  
    @property  # теперь метод как атрибут
    def radius(self):
        return self._radius


    @radius.setter
    def radius(self, value):
        if value < 0:
            raise ValueError("Радиус не может быть отрицательным")
        self._radius = value

# Использование
circle = Circle(5)
print(circle.radius)  # Вывод: 5
circle.radius = 10
print(circle.radius)  # Вывод: 10
```

@classmethod - работает с атрибутами класса (ссылается на класс )
@staticmethod - не связан ни с классом, ни с экземпляром. Он не принимает ни `self`, ни `cls` в качестве первого аргумента.
@property -  позволяет обращаться к методу класса как к атрибуту

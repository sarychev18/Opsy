Менеджеры контекста - это механизм для автоматического открывание/соединения и закрытия/отсоединения от файлом или удаленного ресурса. Даже если отработает исключение файл закроется. Разработчикам помогает правильно открывать и закрывать файлы, вне зависимости сработало исключение или нет.
```python
with open('example.txt', 'r') as file:
    contents = file.read()
    print(contents)
```

Что такое менеджер контекста (контекстный менеджер) ? :: Это механизм для автоматического открытия/присоединения и закрытия/отсоединения от внешнего ресурса, несмотря на возможно ошибки. Помогают снизить вероятность ошибки при работе с внешними ресурсами при помощи ключевого слова with.

## Методы реализации
"Контекстный менеджер можно реализовать тремя способами:

1. Через класс с методами `__enter__` и `__exit__`
2. С помощью `contextlib.contextmanager` и генератора
3. Через `contextlib.ContextDecorator`
```python
# 1. Через класс
class FileManager:
    def __init__(self, filename):
        self.filename = filename
    
    def __enter__(self):
        self.file = open(self.filename, 'w')
        return self.file
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        self.file.close()

# 2. Через генератор
from contextlib import contextmanager

@contextmanager
def file_manager(filename):
    file = open(filename, 'w')
    try:
        yield file
    finally:
        file.close()

# Использование
with FileManager('test.txt') as f:
    f.write('hello')

with file_manager('test.txt') as f:
    f.write('hello')
```
#flashcards/Основы
Первый способ — создать класс с методами __enter__ и __exit__.

__enter__ вызывается при входе в блок with, может вернуть объект для использования после as.

__exit__ вызывается при выходе, принимает три аргумента: exc_type, exc_value и traceback — это информация об исключении, если оно возникло.

Второй способ — использовать декоратор @contextmanager из модуля contextlib.
Пишем функцию-генератор, где код до yield работает как __enter__, а код после yield в блоке finally работает как __exit__.
Значение после yield передаётся в переменную после as."

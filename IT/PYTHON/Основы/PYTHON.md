# Сборщик мусора (Garbage collector)
Сборщик мусора интегрирован в виртуальную машину пайтон и его задача - автоматически освобождать память, удаляя объекты, которые больше не используются программой. Если на объект нет активных ссылок, то сборщик мусора удалит этот объект.

# Динамическая типизация
Пайтон является языком с динамической типизацией данных. Это значит, что переменная может менять свой тип сколько угодно раз во время выполнения программы. Еще тип определяется во время выполнения программы, а не во время объявления переменной
```python
var = 10  # Изначально целочисленное значение
print(type(var))  # Output: <class 'int'>

var = "Hello"  # Теперь строковое значение
print(type(var))  # Output: <class 'str'>

var = [1, 2, 3]  # Теперь список
print(type(var))  # Output: <class 'list'>
```
### Плюсы динамической типизации:
- Гибкость и легкость написания кода: не нужно заранее указывать типы переменных.
- Удобство прототипирования и быстрой разработки: быстрое изменение типов без изменения объявлений.
- Меньшая избыточность кода: переменные легко адаптируются под необходимые типы данных.

### Минусы динамической типизации:
- Потеря безопасности типов: ошибки могут возникать только во время выполнения программы, а не на этапе компиляции.
- Сложность анализа и проверки типов статическими анализаторами.
- Возможное снижение производительности из-за постоянной проверки типов в рантайме.

# Система управления памятью в python

 Работает на оптимизацию и автоматизацию, для того чтобы разработчик больше работал надо логикой программы, а не над управлением памятью.

Система состоит из нескольких механизмов:

- аллокация памяти (Python автоматически выделяет память на созданные объекты

- подсчет ссылок (если 0 то этот объект удаляет сборщик мусора (garbage collector)

- сборщик мусора (удаляет объекты с 0 ссылками и циклическими ссылками)

- менеджер поколений (фильтрует по ссылкам объекты периодически)

- собственный пул аллокаторов (мелки объекты распределяются из специального пула - оптимизирует память)

- контроль над памятью - разработчик сам может влиять на системы управления памятью в пайтон - например с помощью запуска сборщика мусора (garbage collector)

	`gc.collect()` — принудительно запустить сборку мусора.

	`gc.disable()` и `gc.enable()` — временно включать и выключать сборщик мусора.

# GIL
GIL - (global interprner lock) - механизм в интерпретаторе пайтон, позволяющий только одному потоку выполнять байткод в один момент времени, что защищает интерпретатор от конкурентных изменений данных и потенциальных проблем синхронизации.

Пример: Есть 2 покупателя на сайте и они смотрят один и тот же шкаф(последний). Один купил, и вот тут GIL вступает в работу и не дает купить другому шкаф которого нет.

### Как это влияет на производительность.?
Раньше пайтон был однопоточным языком, когда добавили многопоточность, тогда и вынуждены были сделать GIL. 
Есть CPU-запросы - это запросы самого языка во время работы программы. Они всегда ждут своей очереди, перед выполнением своего кода. Что увеличивает время работы программы
Есть еще IO-запросы. Это типо input(), output() - пока ждут ввод-вывод, GIL свободен

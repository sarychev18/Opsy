Исключения - это способ обработки ошибочных ситуаций. Помогает программисту отслеживать ошибки в коде и предотвращать  аварийную остановку программы. 
### Типичные виды исключений в Python:

- **ZeroDivisionError**: деление на ноль.
- **TypeError**: попытка применить некорректную операцию к типу данных.
- **ValueError**: неправильное значение аргумента.
- **FileNotFoundError**: попытки открыть несуществующий файл.
- **IndexError**: выход за пределы индекса коллекции.
- **KeyError**: ключ не найден в словаре.
- **NameError**: использование неопределённой переменной.
- **ImportError**: проблема импорта модулей.

Для создания исключения создают блок кода с try - except.
```python
try:
    x = int(input("Введите целое число: "))
    y = 10 / x
    print(y)
except ValueError:
    print("Ошибка ввода: введено неверное значение")
except ZeroDivisionError:
    print("Деление на ноль невозможно!")
```
Помимо try - except, есть еще else и finally
- **else**: выполняется, если в блоке `try` не возникло исключений.
- **finally**: выполняется независимо от того, было ли исключение или нет.
```python
try:
    x = int(input("Введите число: "))
    y = 10 / x
except ValueError:
    print("Некорректный ввод!")
except ZeroDivisionError:
    print("Деление на ноль недопустимо!")
else:
    print(f"Результат деления: {y}")  # Если всё прошло успешно
finally:
    print("Завершаем программу.")  # Всегда исполняется
```

Что произойдет если сработает исключение ? :: Выведется сообщение, если оно предусмотрено  об ошибке, но программа продолжит работать.

```python
try:
    numerator = float(input("Введите числитель: "))
    denominator = float(input("Введите знаменатель: "))
    result = numerator / denominator
    print(f"Результат деления: {result:.2f}")
except ZeroDivisionError:
    print("Ошибка: Деление на ноль невозможно!")
except ValueError:
    print("Ошибка: Введены неправильные данные!")
```

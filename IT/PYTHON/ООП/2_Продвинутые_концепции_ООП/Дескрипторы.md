Что такое дескриптор ?  :: это класс, который реализует методы `__set__`, `__get__`, `__delete__`. Позволяет не писать много геттеров и сеттеров. Дескрипторы НЕданных это дескриптор, который имеет только метод `__get__`. И наоборот Дескрипторы данных, это когда дескриптор имеет все методы (`__set__`, `__get__`, `__delete__`). Дескрипторов может быть много разных, главное чтобы она отвечал за 1 задачу(принцип SOLID).

**Важно**
- ✅ Дескрипторы выполняют ленивые вычисления
- ✅ Упоминание Данных и не данных
- ✅ Приоритет у дескриптора данных (с `__set__`) выше, чем у дескриптора НЕ данных (только с `__get__`)

```python
class Descriptor:
    def __get__(self, obj, objtype=None):
        print(f"Getting value from {obj}")
        return obj._value
    
    def __set__(self, obj, value):
        print(f"Setting value {value} to {obj}")
        obj._value = value
    
    def __delete__(self, obj):
        print(f"Deleting value from {obj}")
        del obj._value

class MyClass:
    attr = Descriptor()  # Дескриптор как атрибут класса
    
    def __init__(self, value):
        self.attr = value  # Вызовет __set__

obj = MyClass(10)
print(obj.attr)  # Вызовет __get__
del obj.attr     # Вызовет __delete__
```
### Порядок поиска при obj.attr:
1. Data descriptors (с __set__)
2. Атрибуты экземпляра (obj.__dict__)
3. Non-data descriptors (только __get__)
4. Атрибуты класса (type(obj).__dict__)
5. Родительские классы

## ❓ Вопросы для собеседования с ответами

### 1. Чем дескриптор отличается от property?

**Ответ:** Property привязано к одному классу, дескриптор можно переиспользовать в разных классах. Property - это дескриптор под капотом.

### 2. Когда использовать дескриптор вместо property?

**Ответ:** Когда нужно переиспользовать логику доступа к атрибутам в нескольких классах или когда нужен более сложный контроль над жизненным циклом атрибута.

### 3. Что такое `__set_name__` и зачем он нужен?

**Ответ:** Метод вызывается при создании дескриптора, позволяет узнать имя атрибута, к которому он привязан (Python 3.6+). а также добавить в класс, где вызывается дескриптор, имя атрибута.

### 4. Как работают приоритеты при поиске атрибутов?

**Ответ:** Сначала data descriptors, затем атрибуты экземпляра, потом non-data descriptors, затем атрибуты класса и родители.

### 5. Как дескрипторы помогают в тестировании?

**Ответ:** Позволяют легко мокать атрибуты, инжектить зависимости и изолировать логику доступа к данным.

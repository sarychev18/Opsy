Инкапсуляция — это защита "внутренностей" объекта. То есть объект сам управляет своими данными и не позволяет просто так их менять снаружи
Пишется нижнее подчеркивание перед именем метода или переменной
```python
class Box:
    def __init__(self, item):
        self.__item = item  # Прячем содержимое (частная переменная)

    def put_item(self, item):
        self.__item = item  # Меняем игрушку

    def get_item(self):
        return self.__item  # Достаём игрушку


# Использование
box = Box("Мячик")
print(box.get_item())  # Вывод: Мячик

box.put_item("Кукла")
print(box.get_item())  # Вывод: Кукла

print(box.__item)  # ошибка
```
Инкапсуляция в Python работает немного иначе по сравнению с другими языками программирования. Вот основные моменты, которые нужно знать.
#### В пайтоне нет жесткой инкапсуляции
В Python нет строгой защиты переменных, как, например, в Java. Даже если ты пометишь переменную как приватную (например, с двумя подчеркиваниями), к ней всё равно можно получить доступ.
```python
class Box:
    def __init__(self, item):
        self.__item = item  # Приватная переменная


box = Box("Мячик")
print(box._Box__item)  # Получаем доступ: Мячик
```

```python
class Box:
    def __init__(self, item):
        self._inner_item = item  # "Защищённое" содержимое

    def _protected_method(self):
        print("Это защищённый метод")


box = Box("Мячик")
print(box._inner_item)  # Доступ есть, но лучше не трогать
box._protected_method()  # Можно вызвать, но считается не очень хорошей практикой
```
В пайтоне можно нарушить любую инкапсуляцию, однако это требует сознательного решения.

Что такое инкапсуляция в ООП ? :: В Python инкапсуляция реализуется через соглашения об именовании и механизм искажения имён (name mangling). Основные конструкции:<br>1. **Одно подчёркивание `_attr`** — защищённый атрибут (protected). Это соглашение для разработчиков: "атрибут внутренний, не используйте его напрямую вне класса и наследников".<br>2. **Два подчёркивания `__attr`** — приватный атрибут (private). Интерпретатор применяет name mangling: переименовывает `__attr` в `_ClassName__attr`. Это защищает от случайных коллизий при наследовании, но не делает атрибут полностью недоступным.<br>3. **Декоратор `@property`** — основной способ контролируемого доступа. Позволяет превратить методы в атрибуты с логикой валидации при чтении/записи, полностью скрывая внутреннее представление данных.<br>Философия Python: "We are all consenting adults" — инкапсуляция основана на доверии и соглашениях, а не на синтаксических запретах. Прямой доступ к "приватным" атрибутам через `_ClassName__attr` возможен, но не рекомендуется.

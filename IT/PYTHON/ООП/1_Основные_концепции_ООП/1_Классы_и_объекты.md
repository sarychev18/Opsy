ООП - Объектно-ориентированное программирование
ООП реализуется через классы
Класс - это чертеж
Объект - это то что было сделано по этому чертежу
Метод - действие которое умеет делать объект
Параметры(метода) - дополнительная информация, которая передается из вне

```python
class Car: #Создали класс Car
    def __init__(self, param_model, param_color):  #инициализатор имеет 2 переменные
        self.model = param_model #Передаем переменную в сам объект через self
        self.color = param_color #Передаем переменную в сам объект через self

    def drive(self): #Метод класса Car
        print(f'Машина {self.model}, которая имеет {self.color} едет!')

car1 = Car('BMW', 'Grey') #Переменный, которые передаются в объект
car1.drive() #Вызываем метод .drive()
```

Для чего нужно ООП :
• Модульность. - код делиться на классы - делает его организованным и понятным
• Многоразовость -  Наследование и объекты позволяют использовать код многократно.  
• Инкапсуляция - Скрывает данные и защищает их от неправильного использования.
• Полиморфизм - Один интерфейс для работы с разными объектами.
• Упрощает исправление и улучшение кода.
• Моделирование реального мира. Удобно описывать данные и действия (отражать логику реальных объектов).

# Создание класса без слова `class`
**Класс можно создать без ключевого слова `class` с помощью встроенной функции `type()`**. Функция `type()` с тремя аргументами (имя, кортеж базовых классов, словарь атрибутов) динамически создает новый класс. Это основа метаклассов в Python и демонстрирует, что классы сами являются объектами (экземплярами метаклассов).

Пример:
```python
# Создание класса через type()
MyClass = type('MyClass', (), {'x': 42})

# Эквивалентно:
# class MyClass:
#     x = 42

obj = MyClass()
print(obj.x)  # 42
print(type(obj))  # <class '__main__.MyClass'>
```

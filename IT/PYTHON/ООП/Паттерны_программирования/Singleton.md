**Singleton** - это порождающий паттерн проектирования, который гарантирует, что у класса есть только один экземпляр, и предоставляет глобальную точку доступа к этому экземпляру. В Python есть несколько способов реализации: через модуль, метакласс, декоратор, или переопределение `__new__`. Важно учитывать потокобезопасность в многопоточных приложениях.

Обязательно упомяните: единственный экземпляр, глобальная точка доступа, метаклассы, потокобезопасность, альтернатива - модули

## Пример: Реализация через модуль (самый простой способ)
```python
# config.py
class Config:
    def __init__(self):
        self.database_url = "postgresql://localhost:5432/mydb"
        self.api_key = "secret_key"
        self.debug = True

# Создаем единственный экземпляр
config = Config()

# В другом файле
from config import config
print(config.database_url)  # Всегда один и тот же объект
```

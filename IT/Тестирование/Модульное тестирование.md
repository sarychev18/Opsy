#flashcards/Тестирование
что такое модульное тестирование ?  :: Тестирование минимальных единиц кода (функций, методов) в изоляции от внешних зависимостей (БД, сеть)<br><br>**Преимущества:**<br>Можно менять код, не боясь сломать логику.<br>Тесты показывают, как _должен_ использоваться код.<br>TDD заставляет писать модульный код (низкая связность).<br><br>**Риски без тестов:**<br>Исправление одного бага создаёт два новых.<br>Невозможность обновлять зависимости или библиотеки.

**Подтермины**
- TDD - Разработка через тестирование

```python
def test_calculate_total():  # ✅ Префикс test_
    result = calculate_total(100)
    assert result == 100  # ✅ Ключевое слово assert
```

### Фикстуры

```python
import pytest

@pytest.fixture(scope="function")
def db_session():
    session = create_db_connection()  # Setup
    yield session
    session.close()                   # Teardown
    
def test_user_create(db_session):
    # db_session автоматически передан сюда
    pass
```
**Важно**
• Teardown в фикстуре нужен для **очистки ресурсов** после завершения теста (закрытие соединений с БД, удаление временных файлов, откат транзакций)
• Без teardown тесты перестают быть **изолированными**: данные от одного теста влияют на другой (side effects), что приводит к «flakey» (нестабильным) тестам

### Проверка исключений
- Для проверки исключений в `pytest` используется контекстный менеджер `pytest.raises(ExpectedException)`.
- Он гарантирует, что тест упадет, если исключение **не** было выброшено.
```python
import pytest

def test_negative_order():
    # Ожидаем, что внутри блока возникнет ValueError
    with pytest.raises(ValueError):
        create_order(amount=-10)
```
- **Если ошибка возникла:** Тест ✅ (PASS).
- **Если ошибки нет:** Тест ❌ (FAIL) — `Failed: DID NOT RAISE <class 'ValueError'>`.
